Add an Admin User (When Enabling Auth) üîê

  1. Start without security.authorization first (or start with --setParameter enableLocalhostAuthBypass=1 during initial setup).
  2. In mongosh: 
    use admin

  user: "root",
  pwd:  "StrongPassword!123",
  roles: [ { role: "root", db: "admin" } ]
})
  3. Now enable in D:\mongodb\config\mongod.cfg: 
    security:
  authorization: enabled
  4. Restart MongoDB, then connect: 
    mongosh -u root -p --authenticationDatabase admin
üß† Use a unique, strong password. For projects, create app-specific users with minimal roles instead of reusing root.

Backups & Upgrades üß≥

  • Back up D:\mongodb\data regularly (stop service first for a consistent snapshot, or use mongodump).
  • To upgrade MongoDB:
    1. Stop service: net stop MongoDB
    2. Install new version (to the same D:\mongodb\bin or a versioned folder like D:\mongodb\bin-8.0).
    3. Keep using the same dbPath in D:\mongodb\data.
üì¶ Logical backup example:
